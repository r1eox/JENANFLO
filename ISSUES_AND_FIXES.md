# JenanFlo - Issues and Fixes

## 1. ملخص عام
المشروع يعمل محليًا مع قاعدة بيانات SQLite عبر Prisma، ولكن يوجد عدد من النواقص التقنية والتنظيمية التي تحتاج إصلاح:
- بقايا من إعدادات MongoDB وSupabase غير مستخدمة.
- تبعيات غير ضرورية موجودة في `package.json`.
- واجهة `README.md` غير محدثة لحالة المشروع الحالية.
- بعض APIs غير آمنة أو غير مكتملة.
- بعض صفحات الواجهة تعتمد على `localStorage` أو `cookies` بدون حماية كافية.

## 2. توثيق ناقص
- يجب تحديث `README.md` ليشمل:
  - الاعتمادات الأساسية: Next.js، Prisma، SQLite، bcryptjs.
  - كيفية تشغيل المشروع محليًا: `npm install`, `npm run dev`.
  - شرح أن قاعدة البيانات تستخدم `dev.db` ويتم تجاهلها عبر `.gitignore`.
  - أن البيئة تدعم `.env.local` و`.env`.
  - ملاحظات حول عدم رفع مفاتيح وأسرار.

## 3. تنظيف التبعيات والملفات غير المستخدمة
### يجب حذف أو إعادة تقييم:
- `mongoose` وملفات `src/models/*.ts`
- `@supabase/supabase-js` وملف `src/lib/supabase.ts`
- `@prisma/adapter-pg` و`pg` إذا لم يعد هناك استخدام فعلي لقاعدة Postgres
- `src/lib/mongodb.ts`
- `src/lib/mockData.ts` إذا لم يعد يستخدم

## 4. قضايا API رئيسية
### 4.1 `src/app/api/users/route.ts`
- ينشئ مستخدمًا باستخدام كلمة مرور عادية في POST.
- يجب استخدام `bcrypt.hash` أو حظر إنشاء المستخدم بهذه الطريقة في API غير الآمن.

### 4.2 `src/app/api/auth/login/route.ts`
- يحتوي على حساب admin ثابت داخل الكود.
- يجب نقل بيانات الاعتماد إلى `.env.local` أو جعله يعمل عبر قاعدة البيانات.
- الكوكي يتم تعيينه كـ `httpOnly: false` رغم أنه حساس.

### 4.3 `src/app/api/orders/route.ts`
- `markCouponUsed` لا تشير إن كان الكوبون صالحًا أو متجاوز الحد.
- يجب أن تعطي ردًا واضحًا إذا كان الخصم غير صالح بدلًا من قبول الطلب silently.

### 4.4 `src/app/api/coupons/route.ts`
- لا يوجد دعم لتحديث تفاصيل الكوبون مثل الكود أو الخصم أو رقم العميل.
- يمكن إضافة PATCH أوسع مع حقل `active` و`usageLimit`.

### 4.5 `src/lib/localDb.ts`
- الكثير من `any` و`Partial` مما يضعف السلامة.
- يحتاج تعزيز التحقق من صحة المدخلات قبل إنشاء/تحديث الكائنات.
- `getCouponByCode` يعامل الكوبون غير الصالح كـ `undefined` فقط، بدون رسالة مفيدة.

## 5. واجهة المستخدم والنواقص
### 5.1 `src/app/checkout/page.tsx`
- يعتمد بقوة على `localStorage` لحياة المستخدم وجلب السلة.
- لا يوجد فحص قوي لحالات `deliveryToOther` أو بيانات المستلم.
- قد يحتاج تحسين تدفق تطبيق الكوبون وتحديث السعر النهائي.

### 5.2 `src/app/admin/coupons/page.tsx`
- لا يوجد تحرير لكوبون موجود حتى الآن.
- لا يوجد عرض رسالة حالة تحميل واضحة عند استدعاء API.
- لا يوجد التحقق من أخطاء الشبكة أو رسائل الخطأ التفصيلية.

### 5.3 `src/app/products/[id]/page.tsx`
- جيد كخيار fallback، لكنه يعتمد على `localProducts` أولا ثم API.
- إذا كان اسم المنتج أو ID غير موجود، يمكن تحسين واجهة الخطأ.

## 6. أمان النظام
- تخزين معلومات المستخدم في `localStorage` وملف الكوكي `jenanflo_user` غير محمي.
- لا يوجد جلسات مسجلة أو مصادقة حقيقية لصفحات `admin` سوى middleware بسيط.
- يجب عدم ترك بيانات الاعتماد `admin@jenanflo.com / admin123` داخل الكود.

## 7. توصيات سريعة للتعديل
### 7.1 تحديث `README.md`
- وصف المعمارية الحالية.
- كيفية تثبيت الحزم وتشغيل المشروع.
- توضيح قاعدة البيانات المحلية.

### 7.2 تنظيف الحزم
- إزالة `mongoose`, `@supabase/supabase-js`, `@prisma/adapter-pg`, `pg`, `@types/pg` من `package.json` إذا لم يتم استخدامها.

### 7.3 تحسين API
- جعل `POST /api/users` آمنًا أو تقييده.
- تخزين بيانات admin في env أو DB.
- تحسين `markCouponUsed` لتعيد حالة واضحة.
- دعم تحديث الكوبونات بشكل كامل.

### 7.4 تحسين تجربة المستخدم
- إضافة رسائل حالة تحميل وأخطاء أفضل في واجهات الإدارة.
- تحسين حقل `couponCode` في صفحة الدفع ومعالجة الأخطاء.
- دعم عناوين محفوظة بشكل أفضل عبر `userStorage`.

## 8. خطوتي التالية المقترحة
1. تنظيف الحزم والملفات غير المستخدمة.
2. تحديث `README.md` تلخيصًا للحالة الحالية.
3. تحسين API الخاص بـ `users`, `auth`, `orders`, `coupons`.
4. إصلاح واجهات الإدارة والصفحة الرئيسية بالكوبونات.
